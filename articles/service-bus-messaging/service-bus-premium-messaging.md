# Уровни обмена сообщениями через служебную шину Premium и Standard 

Обмен сообщениями через служебную шину включает такие сущности обмена сообщениями, как очереди и темы, и объединяет возможности корпоративного обмена сообщениями с расширенной семантикой публикации и подписок в масштабе облака. Обмен сообщениями через служебную шину используется в качестве средства коммуникации во многих сложных облачных решениях.

Обмен сообщениями через служебную шину уровня *Premium* выполняет распространенные просьбы клиентов по поводу масштаба, производительности и доступности критически важных приложений. Несмотря на практически идентичные наборы функций, эти два уровня обмена сообщениями через служебную шину предназначены для использования в разных условиях.

Некоторые высокоуровневые различия представлены в приведенной ниже таблице.

| Premium | Стандартная |
|---------------------------------------|--------------------------------|
| Высокая пропускная способность | Переменная пропускная способность |
| Прогнозируемая производительность | Переменная задержка |
| Предсказуемое ценообразование | Переменная оплата по мере использования |
| Возможность масштабирования в зависимости от рабочей нагрузки | Недоступно |
| Размер сообщения > 256 КБ | Размер сообщения равен 256 КБ |

**Обмен сообщениями через служебную шину уровня "Премиум"** обеспечивает изоляцию ресурсов на уровне процессора и памяти, так что рабочая нагрузка каждого клиента выполняется изолированно. Контейнер ресурса называется *единицей обмена сообщениями*. Для каждого премиального пространства имен выделяется хотя бы одна единица обмена сообщениями. Для каждого пространства имен служебной шины Premium можно приобрести одну, две или четыре единицы обмена сообщениями. Одна единица рабочей нагрузки или сущность может охватывать несколько единиц обмена сообщениями, количество которых можно менять; при этом стоимость услуг рассчитывается по 24-часовому или суточному тарифу. В результате вы получаете решение на базе служебной шины с прогнозируемой и повторяемой производительностью.

Оно отличается не только более предсказуемой производительностью и высокой доступностью, но и более высокой скоростью работы. Служба обмена сообщениями через служебную шину уровня "Премиум" работает на основе подсистемы хранилища, реализованной в [концентраторах событий Azure](https://azure.microsoft.com/services/event-hubs/). Уровень Premium обеспечивает более высокую скорость передачи данных при пиковой нагрузке, чем уровень Standard.

## Технические отличия обмена сообщениями уровня Premium

Ниже представлены некоторые различия между уровнями обмена сообщениями Premium и Standard

### Секционированные очереди и разделы

Секционированные очереди и разделы поддерживаются на уровне "Премиум", но работают не так, как на уровнях обмена сообщениями через служебную шину уровня "Стандартный" и "Базовый". Обмен сообщениями уровня Premium не использует SQL как хранилище данных и не участвует в борьбе за ресурсы, которая может быть связана с общей платформой. В результате необходимость в секционировании отпадает. Кроме того, для обмена сообщениями на уровне "Стандартный" используется 16 разделов, а на уровне "Премиум" — два. Два раздела обеспечивают высокую доступность и больше подходят для среды выполнения "Премиум". Дополнительные сведения о секционировании см. в статье [Секционированные сущности обмена сообщениями](service-bus-partitioning.md).

### Экспресс-сущности

Поскольку обмен сообщениями уровня "Премиум" работает в полностью изолированной среде, экспресс-сущности не поддерживаются в пространстве имен уровня "Премиум". Дополнительные сведения о функции экспресс см. в разделе о свойстве [Microsoft.ServiceBus.Messaging.QueueDescription.EnableExpress](https://msdn.microsoft.com/library/azure/microsoft.servicebus.messaging.queuedescription.enableexpress.aspx).
